# Linux Environment Setup & Exploration

## Overview

This repository documents **Task 1: Linux Environment Setup & Exploration**, completed as part of the DevOps Internship program.  
The objective of this task is to gain hands-on experience with Linux fundamentals, including system navigation, file management, permissions, and basic system monitoring using the terminal.

---

## Environment Details

- **Operating System:** Ubuntu 24.04 (via WSL on Windows)
- **Access Method:** Terminal (CLI)
- **Tools Used:**
  - Core Linux utilities (`ls`, `cd`, `pwd`, `mkdir`, `rm`, etc.)
  - Text editors (`nano`, `vim`)
  - System monitoring tools (`top`, `df -h`, `free -m`)

---

## Task Objectives

- Install and access a Linux environment
- Understand Linux directory structure
- Perform basic file and directory operations
- View and edit files using terminal-based editors
- Manage file permissions and ownership
- Monitor system resources
- Document commands and outputs clearly

---

## Commands Practiced

- **Navigation:** `pwd`, `ls`, `cd`
- **File & Directory Management:** `mkdir`, `touch`, `rm`, `rmdir`
- **File Viewing & Editing:** `cat`, `less`, `nano`, `vim`
- **Permissions & Ownership:** `ls -l`, `chmod`, `chown`
- **System Monitoring:** `top`, `df -h`, `free -m`

All executed commands and their outputs are recorded for reference.

---


---

## Key Learnings

- Difference between **root user** and **normal user**
- Importance and usage of **file permissions**
- Practical understanding of Linux filesystem hierarchy
- How Linux handles permission-related errors
- Basic system resource monitoring from the command line

## Outcome

This task strengthened my confidence in navigating and managing Linux systems, which is a foundational skill for DevOps and cloud-related roles.
